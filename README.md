# 🦖 DIPLO-DOC: Реестр услуг благополучателей

Интеллектуальная система учёта услуг, построенная на мощных скалах **CodeIgniter 4** (PHP 8.2) и **Angular 17** (Signals & Standalone). Проект оптимизирован под работу с большими реестрами (1000+ записей) и усилен нейросетевыми ассистентами.

## 🧬 Архитектурные особенности (Seniors only)
*   **AI-Powered Registry:** Внедрена система **Natural Language to Filter**. Поиск понимает человеческие запросы (напр. *"консультации Иванова за прошлую неделю"*), преобразуя их в SQL-фильтры через LLM.
*   **Smart Excel Import:** Реализована пакетная обработка (Chunks) по 200 строк с **AI-коррекцией**. Система автоматически исправляет опечатки в статусах и типах услуг, сопоставляя их со справочниками.
*   **Staging Import Pattern:** Использование временных (Temporary) таблиц PostgreSQL для гарантированной атомарности и целостности данных при массовой загрузке.
*   **Signals State Management:** Фронтенд на Angular 17 использует `Signals` для реактивного управления состоянием таблиц, фильтров и процесса загрузки.
*   **Distributed Tracing:** Внедрен сквозной `X-Trace-Id`. Любой запрос с фронта можно отследить в логах бэкенда по уникальной метке.
*   **Dino-Logging System:** Кастомный хелпер логирования с эмодзи-маркерами для визуального мониторинга "метеоритов" (ошибок) и "следов" (AI-действий).

## 🌋 Быстрый старт (Раскопки)

### 1. Бэкенд (CodeIgniter 4)
Находится в папке `/backend`.
1.  **Настройка окружения:** Скопируйте `env` в `.env` и настройте подключение к **PostgreSQL 15+**.
2.  **Зависимости:**
    ```bash
    composer install
    ```
3.  **Миграции и наполнение (Seeders):** Создает структуру таблиц и эталонные справочники.
    ```bash
    php spark migrate
    php spark db:seed MainSeeder
    ```
4.  **Запуск сервера:**
    ```bash
    php spark serve
    ```

### 2. Фронтенд (Angular 17)
Находится в папке `/frontend`.
1.  **Установка пакетов:** `npm install`
2.  **Запуск:** `ng serve` (Интерфейс: `http://localhost:4200`)

## 📜 Основные эндпоинты (API)
Полная спецификация — в `swagger.json`.
*   `GET /api/services` — Реестр с пагинацией, **серверной сортировкой** и поиском.
*   `POST /api/import` — Умный импорт Excel (Multipart/form-data) с отчетом об AI-правках.
*   `GET /api/export` — Экспорт текущего отфильтрованного вида в `.xlsx`.
*   `POST /api/ask` — Эндпоинт для AI-ассистента Диплодока.

## 🐾 Принятые решения (Ход мысли)
1.  **Autocomplete UX:** Реализован живой поиск бенефициаров с `debounceTime(300)`, чтобы комфортно работать с базой в 1000+ человек.
2.  **Excel Templates:** Шаблон для импорта генерируется динамически с **Data Validation** (выпадающие списки внутри ячеек Excel), минимизируя ошибки ввода.
3.  **Soft Deletes:** Реализовано "мягкое удаление". Услуги не стираются физически, а архивируются (ТЗ п.6).
4.  **Batch AI Processing:** Для экономии токенов и скорости, исправление ошибок в Excel происходит одним пакетным запросом к LLM.
